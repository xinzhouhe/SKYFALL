# SKYFALL — Dynamic Flipping Integration

This directory contains the SKYFALL LEO satellite bottleneck attack simulator (NDSS 2025),
extended with a third topology: **Dynamic Flipping** (from "Three Links Are Not a Grid").

## Three topology modes

| Mode | Script | Data dir | ISLs/sat | Routing |
|---|---|---|---|---|
| +Grid | `generate_flow_grid.py` | `+grid_data/` | 4 (fixed) | Manhattan `find_next_sat` |
| Circle | `generate_flow_circle.py` | `circle_data/` | 2 (intra only) | `cycle_floyd` |
| Dynamic Flipping | `generate_flow_flipping.py` | `flipping_data/` | 2–3 (time-varying) | BFS per frame |

---

## Setup: generate topology.json first

Before running any flipping scripts, generate `topology.json` in the repo root:

```bash
# Step 1: write SKYFALL's Walker constellation as TLE
cd skyfall/
python write_skyfall_tle.py --output ../skyfall_walker.tle

# Step 2: run CDN topology generator (requires dynamic-flipping-topology repo)
cd ../../dynamic-flipping-topology/
python generate.py \
    --tle ../SKYFALL/skyfall_walker.tle \
    --start "2020-06-01 00:00:00" \
    --end   "2020-06-01 01:30:00" \
    --step  15 \
    --output ../SKYFALL/topology.json
```

`topology.json` path is configured in `config.json` → `"topology_json"` field.

## Running the flipping pipeline

Same invocation pattern as +Grid:

```bash
# Generate satellite positions (same as always — shared across all 3 topologies)
python generate_lla.py 3600

# For each timeslot, generate traffic (replace N with timeslot index)
python generate_flow_flipping.py N

# Attack analysis
python time_slot_analysis_flipping.py N <ratio>

# Aggregate across timeslots
python aggregated_deployment_flipping.py <total_time> <ratio>
```

## Key design decisions

### Satellite indexing
SKYFALL index: `sat_index = orbit_id * 22 + sat_id` (0–1583).
`write_skyfall_tle.py` writes TLEs in this exact order, so CDN satellite index k == SKYFALL
satellite index k. No remapping needed anywhere.

### Timeslot → CDN frame mapping
CDN frames are 15-second intervals; SKYFALL timeslots are 1-second.
Mapping: `frame_idx = timeslot // 15` (constant `CDN_STEP = 15` in all flipping scripts).
The last CDN frame is reused for any timeslot beyond the topology window.

### Link traffic accounting
`generate_flow_flipping.py` does NOT use a `sat * 6 + offset` flat array (that assumes
exactly 4 ISLs in fixed positions). Instead:
- `link_traffic_isl[]` is sized to the number of edges in the current frame
- `link_id_map[(a, b)]` maps directed edge to index into that array
- Satellites in re-acquisition (0 inter-orbit ISLs) simply have no inter-orbit entry
- GSL traffic (`uplink_traffic[]`, `downlink_traffic[]`) stays per-satellite as before

### Routing
`find_next_sat()` / `floyd()` in grid code uses Manhattan distance on a regular 72×22 grid.
Flipping uses BFS (`bfs_next_hop`) over the loaded adjacency — topology-agnostic and correct
for any irregular or time-varying graph.

### Candidate satellite ranking in attack analysis
`time_slot_analysis_flipping.py` replaces grid Manhattan hop-count ranking with BFS hop
distance from the target satellite, over the current frame's adjacency.

## Common issues

**`topology.json` not found**: Run `write_skyfall_tle.py` + CDN `generate.py` first.
Check the `"topology_json"` path in `config.json`.

**CDN satellite count mismatch**: CDN will find 72 planes with exactly 22 sats each on the
regular Walker constellation. If `assign_orbits` produces a different count, verify the TLE
epoch matches the CDN `--start` time and that `--n-planes 72` is set.

**Frame index out of range**: CDN topology window must cover all simulated timeslots.
For 3600 timeslots at CDN_STEP=15, you need at least 240 CDN frames (60 minutes).
